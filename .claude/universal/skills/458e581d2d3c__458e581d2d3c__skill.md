# Sprint Initialization Protocol

You are the Master Orchestrator. Run a structured sprint kickoff.

---

## Phase 1 — Platform Baseline Snapshot

Gather current state before planning:

```bash
# Test baseline
cd backend && python -m pytest --tb=no -q 2>&1 | tail -3

# TypeScript
cd frontend && npx tsc --noEmit 2>&1 | tail -3

# Open issues
git log --oneline -20

# Uncommitted changes
git status --short
```

## Phase 2 — Backlog Review

Ask the user:
1. **What's the sprint goal?** (e.g., "Enable live trading", "Improve AI accuracy", "Launch new feature X")
2. **What's the time horizon?** (1 week / 2 weeks / 1 month)
3. **Any P0 blockers** from the last sprint that must be fixed first?
4. **Which agents should lead?** (trading, infrastructure, AI, frontend, security)

## Phase 3 — Task Decomposition

Based on the sprint goal, decompose into phases:

```
SPRINT GOAL: [user's goal]
DURATION: [N weeks]

PHASE 0 — Blockers (Day 1-2)
  - [P0 items that block everything else]
  - Agent: [which specialist]

PHASE 1 — Core (Day 2-5)
  - [primary deliverables]
  - Agent: [which specialist]

PHASE 2 — Enhancement (Day 5-8)
  - [secondary improvements]
  - Agent: [which specialist]

PHASE 3 — Testing & Hardening (Day 8-10)
  - Full test suite run
  - Pre-deploy checks
  - Iron Gate review
```

## Phase 4 — Quality Gates

Define what "done" means for this sprint:

| Gate | Criteria |
|------|----------|
| Tests | ≥ baseline passed, 0 new failures |
| TypeScript | 0 errors |
| Coverage | ≥ 70% backend, ≥ 60% frontend |
| API Health | "healthy" + no degraded mode |
| Security | 0 CRITICAL CVEs |
| Schema | No drift — all ALTER TABLEs run before deploy |

## Phase 5 — Agent Assignments

Route sprint tasks to appropriate agents:
- **Trading/Quant work** → `apex-quant-architect`
- **Infrastructure/Deploy** → `apex-coordinator` + `reliability-security-sentinel`
- **AI/ML work** → `apex-model-trainer` + `intelligence-lead`
- **Frontend** → `product-experience-engineer`
- **Code quality** → `fintech-integrity-auditor`
- **Security gate** → `reliability-security-sentinel`
