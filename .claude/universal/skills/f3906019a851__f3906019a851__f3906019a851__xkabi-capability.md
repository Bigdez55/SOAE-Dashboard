# GEN.OS XKABI Capability — Definition & Registration

Add a new XKABI capability right to the GEN.OS security model.
Reference: `kernel/xenos/include/xkabi_rights.h` + `kernel/xenos/cap/xkabi_capabilities.c`

## Step 1 — Define the Right

Edit `kernel/xenos/include/xkabi_rights.h`:

```c
/* Existing pattern — follow exactly: */
#define XKABI_RIGHT_[DOMAIN]_[ACTION]   (1ULL << [N])
/* Human-readable alias (used in .gsd files): */
#define XK_RIGHT_[DOMAIN]_[ACTION]      XKABI_RIGHT_[DOMAIN]_[ACTION]
```

Rules:
- Bit positions are permanent — never reuse a retired bit
- Bit N where N = next available bit (check current file for highest)
- Domain prefix examples: `NET`, `STORAGE`, `DEVICE`, `CRYPTO`, `PROC`, `AUDIT`
- Action: `SEND`, `RECV`, `READ`, `WRITE`, `SPAWN`, `MOUNT`, `KEY`
- Add a comment: `/* [One-line: what this right grants, since Sprint N] */`

## Step 2 — Register in Capability Enforcement

Edit `kernel/xenos/cap/xkabi_capabilities.c`:

```c
/* In the capability descriptor table: */
static const xkabi_cap_desc_t g_cap_descriptors[] = {
    /* Existing entries ... */
    {
        .right       = XKABI_RIGHT_[DOMAIN]_[ACTION],
        .name        = "XK_RIGHT_[DOMAIN]_[ACTION]",
        .description = "[What this right grants]",
        .audit_level = XKABI_AUDIT_[LEVEL],  /* NONE / LOG / ALERT */
        .introduced  = [N],                   /* Sprint number */
    },
};
```

## Step 3 — Update CAP_NAMES Table

In `init/gensd/service.c`, the `CAP_NAMES` table maps string names to values for .gsd parsing:

```c
/* Find the CAP_NAMES[] array and add: */
{ "XK_RIGHT_[DOMAIN]_[ACTION]", XK_RIGHT_[DOMAIN]_[ACTION] },
```

## Step 4 — Security Review (guardian-sentinel)

Before merging, the guardian-sentinel agent must confirm:

- [ ] Principle of least privilege: right is narrowly scoped
- [ ] No service can bypass this right via another vector
- [ ] Audit level appropriate (sensitive ops -> XKABI_AUDIT_ALERT)
- [ ] Right bit does not conflict with existing bits
- [ ] .gsd parser table updated (step 3)
- [ ] No raw hex in any .gsd file — only `XK_RIGHT_*` alias

## Step 5 — Write a Test

```c
/* In tests/security/test_xkabi_barriers.c (or new file): */
static void test_[right]_isolation(void)
{
    /* Verify a context WITHOUT the right cannot perform the action */
    xkabi_context_t ctx_no_right = { .rights = 0 };
    int rc = xkabi_check(&ctx_no_right, XKABI_RIGHT_[DOMAIN]_[ACTION]);
    assert(rc == XKABI_ERR_EPERM);

    /* Verify a context WITH the right can perform the action */
    xkabi_context_t ctx_with_right = { .rights = XK_RIGHT_[DOMAIN]_[ACTION] };
    rc = xkabi_check(&ctx_with_right, XKABI_RIGHT_[DOMAIN]_[ACTION]);
    assert(rc == 0);

    pal_console_printf("test_[right]_isolation: PASS\n");
}
```
