# API Debugging Protocol

Trace a failing endpoint from HTTP response back to root cause.

---

## Step 1 — Reproduce the Error

```bash
TOKEN=$(curl -s -X POST https://api.elsontrade.com/api/v1/auth/login \
  -H "Content-Type: application/json" \
  -d '{"email":"claude.test@example.com","password":"ElsonTest2026!"}' \
  | python3 -c "import sys,json; print(json.load(sys.stdin).get('access_token',''))")

# Replace METHOD, PATH, and BODY with the failing request
curl -sv -X METHOD https://api.elsontrade.com/api/v1/PATH \
  -H "Authorization: Bearer $TOKEN" \
  -H "Content-Type: application/json" \
  -d 'BODY' 2>&1 | grep -E "^[<>]|HTTP|{|}"
```

Note the HTTP status code and response body.

## Step 2 — Common Status Code Diagnosis

**404 Not Found:**
```bash
# Check if route is registered
grep -rn "PATH\|router\." backend/app/api/ --include="*.py" | grep -i "path_component" | head -10
# Check api.py includes the router
grep -n "include_router\|PATH_PREFIX" backend/app/api/api_v1/api.py | head -20
```

**422 Unprocessable Entity:**
```bash
# Check Pydantic schema for the request body
grep -rn "class.*Request\|class.*Schema" backend/app/schemas/ --include="*.py" | head -10
# The response body will show which field failed validation
```

**401 Unauthorized:**
```bash
# Check if endpoint has auth dependency
grep -rn "Depends\|oauth2_scheme\|get_current_user" backend/app/api/api_v1/endpoints/ --include="*.py" | grep -i "endpoint_name" | head -5
# Re-verify token is valid
echo $TOKEN | python3 -c "
import sys, base64, json
token = sys.stdin.read().strip()
parts = token.split('.')
if len(parts) == 3:
    payload = base64.b64decode(parts[1] + '==')
    print(json.loads(payload))
"
```

**500 Internal Server Error:**
```bash
# Check recent logs for traceback
gcloud logging read \
  'resource.type="cloud_run_revision" AND resource.labels.service_name="elson-backend" AND severity>=ERROR' \
  --project=elson-33a95 \
  --limit=10 \
  --format="table(timestamp,textPayload)" 2>/dev/null | head -30
```

## Step 3 — Trace the Code Path

```bash
# Find the endpoint handler
grep -rn "def.*endpoint_function\|@router\.\(get\|post\|put\|delete\).*path" \
  backend/app/api/ --include="*.py" | grep "path_component" | head -5

# Find the service it calls
grep -rn "service\.\|await.*service" backend/app/api/api_v1/endpoints/ --include="*.py" \
  | grep "function_name" | head -5
```

## Step 4 — Database Layer Check

```bash
# Check if model/table exists
grep -rn "class.*Model\|__tablename__" backend/app/models/ --include="*.py" | head -20

# Check for common DB issues
grep -rn "Column(Float\|relationship.*back_populates" backend/app/models/ --include="*.py" \
  | head -10
```

## Step 5 — Pydantic Response Model Mismatch

```bash
# Find response_model for the endpoint
grep -n -A2 "@router\.\(get\|post\)" backend/app/api/api_v1/endpoints/ENDPOINT.py \
  | grep "response_model" | head -10

# Check if actual return matches
# ResponseValidationError = return dict doesn't match response_model
# Fix: change response_model=Dict[str,Any] or fix the return value
```

## Step 6 — Local Reproduction (if accessible)

```bash
cd backend

# Quick local test without full server
python3 -c "
import asyncio
from app.api.api_v1.endpoints.ENDPOINT import FUNCTION
# or test the service directly
from app.services.SERVICE import ServiceClass
" 2>&1 | head -20
```

## Debug Report

```
API DEBUG REPORT
══════════════════════════════
Endpoint:      METHOD /api/v1/PATH
HTTP Status:   [404 / 422 / 401 / 500]
Error Message: [from response body]

Root Cause:    [missing route / schema mismatch / auth / DB error / exception]
Location:      [file:line]

Fix:           [description]
  File: backend/app/[path]
  Change: [what to change]
```

**Common fixes by status:**
- 404 → Add route to `api.py` `include_router` or missing `@router.post`
- 422 → Fix Pydantic schema field names/types or `response_model=Dict[str,Any]`
- 401 → Check `Depends(get_current_user)` or token expiry
- 500 → Check Cloud Run logs for full traceback; check DB connection and schema
