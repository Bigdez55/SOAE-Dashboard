# Sprint 3 — XPKG Package Manager Design & Scaffold

XPKG is the GEN.OS package manager. Manages signed package install/remove/update.
Depends on: XSEC (signing/verification), XNET (download), XSTORE (local DB).

## Architecture

```
pkg/xpkg/
├── include/
│   ├── xpkg.h          # Public API
│   └── xpkg_types.h    # xpkg_manifest_t, xpkg_entry_t, xpkg_state_t
├── xpkg_core.c         # Init, package registry, state machine
├── xpkg_manifest.c     # Package manifest parser (.gpkg format)
├── xpkg_fetch.c        # Download via XNET, progress tracking
├── xpkg_verify.c       # Signature verification via XSEC
├── xpkg_install.c      # Rootfs overlay + service registration
└── xpkg_db.c           # Local package database (SQLite-free binary format)
```

## .gpkg Manifest Format

```ini
[Package]
Name = [package]
Version = 1.0.0
Arch = x86_64
Description = [One-line description]

[Dependencies]
Requires = [dep-a] >= 1.0, [dep-b]

[Files]
/opt/genos/services/[binary] = sha256:[hash]
/etc/gensd/services/[service].gsd = sha256:[hash]

[Signing]
# Ed25519 public key fingerprint of the GEN.OS package signing key
KeyFingerprint = [hex]
Signature = [base64-encoded Ed25519 signature over manifest body]
```

## Design Constraints

| Constraint | Specification |
|------------|--------------|
| Package signing | Ed25519 (via XSEC) — reject any unsigned package |
| DB format | Binary flat-file (no SQLite, no libc FILE*) |
| Memory | Static package registry (256 slots). No heap. |
| Atomic install | Write to temp path, verify SHA-256, rename — no partial state |
| XKABI rights | `XK_RIGHT_STORAGE_WRITE`, `XK_RIGHT_NET_RECV`, `XK_RIGHT_CRYPTO_KEY` |
| Rollback | Keep previous version in `/var/genos/pkg/rollback/[name]/` |
