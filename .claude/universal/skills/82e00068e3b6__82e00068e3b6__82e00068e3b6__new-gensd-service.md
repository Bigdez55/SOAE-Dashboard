# GEN.OS New GENSD Service — GSD Descriptor + Registration

Registers a new service with the GENSD init system.
Reference: `init/gensd/service.c` (parser), `init/gensd/supervisor.c` (lifecycle).

## GSD File Format

File: `init/gensd/services/[service].gsd`

```ini
[Unit]
Name = [service]
Description = [One-line human description]
# Comma-separated list of .gsd files this service depends on
Requires = xstore.gsd, xnet.gsd
# Startup ordering (subset of Requires)
After = xstore.gsd

[Service]
# Type: daemon (long-running) | oneshot (runs to completion)
Type = daemon
# Absolute path to binary in GEN.OS rootfs
Exec = /opt/genos/services/[service]
# RestartOn: failure | always | never
RestartOn = failure
RestartDelay = 2s

[XKABI]
# XK_RIGHT_* aliases from kernel/xenos/include/xkabi_rights.h
# Pipe-separated. Grant minimum necessary rights.
Capabilities = XK_RIGHT_[A] | XK_RIGHT_[B]
```

## Available XK_RIGHT_* Values

Source: `kernel/xenos/include/xkabi_rights.h`

```
XK_RIGHT_STORAGE_READ     — Read from xstore
XK_RIGHT_STORAGE_WRITE    — Write to xstore
XK_RIGHT_NET_SEND         — Send network packets (XNET)
XK_RIGHT_NET_RECV         — Receive network packets (XNET)
XK_RIGHT_PROC_SPAWN       — Spawn child processes
XK_RIGHT_DEVICE_DRM       — DRM/display access
XK_RIGHT_DEVICE_INPUT     — Input device access
XK_RIGHT_AUDIT_READ       — Read audit log
XK_RIGHT_AUDIT_WRITE      — Write audit events
XK_RIGHT_CRYPTO_KEY       — Cryptographic key operations (XSEC)
```

Never use raw hex values — always use `XK_RIGHT_*` aliases.

## Dependency Graph Check

Before writing `Requires`, verify no circular dependencies:

```bash
# List all existing .gsd files and their After= fields
grep -h "^After" init/gensd/services/*.gsd 2>/dev/null

# Manually trace: [service] → requires [A] → requires [B] → cycle?
# GENSD uses Kahn's algorithm (init/gensd/service.c: gsd_dep_order())
# A cycle will cause GENSD to refuse to start the affected services.
```

## Registration Checklist

- [ ] GSD file placed in `init/gensd/services/[service].gsd`
- [ ] `Name =` matches binary filename (no spaces)
- [ ] `Exec =` path is absolute and matches rootfs layout
- [ ] `Capabilities =` uses ONLY `XK_RIGHT_*` aliases
- [ ] No circular dependency (trace manually, test in QEMU)
- [ ] `guardian-sentinel` reviewed XKABI rights (principle of least privilege)
- [ ] Health endpoint at `/health` returns `{"status": "healthy"}`
- [ ] Corresponding k8s manifest in `platform/k8s/` updated
- [ ] Binary built and placed in `platform/services/[service]/`
