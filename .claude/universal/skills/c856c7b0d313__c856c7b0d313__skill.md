# Database Migration Protocol

**Critical:** Always run ALTER TABLE on Cloud SQL BEFORE deploying code with new columns. Never the reverse.

---

## Step 1 — Detect What Changed

```bash
git diff HEAD -- backend/app/models/ | grep "^\+" | grep "Column("
```

List every new column with its table and type.

## Step 2 — Start Cloud SQL Proxy (if not running)

```bash
# Check if proxy already running
lsof -i :15432 | head -5

# If not running, start it:
cloud_sql_proxy -instances=elson-33a95:us-west1:elson-postgres=tcp:15432 &
sleep 3
```

## Step 3 — Generate Alembic Migration

```bash
cd backend
source venv/bin/activate 2>/dev/null || true
export DATABASE_URL="postgresql://elson_user:PASSWORD@127.0.0.1:15432/elson_db"

# Auto-generate migration from model diff
alembic revision --autogenerate -m "add_[description]_$(date +%Y%m%d)"
```

Review the generated file in `backend/alembic/versions/` before applying.

## Step 4 — Review Generated Migration

```bash
# Show the latest generated migration
ls -t backend/alembic/versions/*.py | head -1 | xargs cat
```

Verify:
- Only `op.add_column()` calls (no unexpected drops)
- Correct table names
- Correct column types (use `sa.Numeric(18,8)` for money, NOT Float)

## Step 5 — Apply to Cloud SQL

```bash
cd backend
alembic upgrade head
```

Or if running a specific migration:
```bash
alembic upgrade +1
```

## Step 6 — Verify Schema Applied

```bash
PGPASSWORD=PASSWORD psql -h 127.0.0.1 -p 15432 -U elson_user -d elson_db \
  -c "\d {table_name}"
```

Confirm new columns appear.

## Step 7 — Safe to Deploy

```bash
echo "Schema updated. Safe to deploy code."
```

---

## Emergency Manual ALTER TABLE

If Alembic fails, run ALTER TABLE directly:

```sql
-- Add column with safe default
ALTER TABLE {table} ADD COLUMN IF NOT EXISTS {column} {type} DEFAULT {default};

-- Example:
ALTER TABLE users ADD COLUMN IF NOT EXISTS risk_tolerance VARCHAR(20) DEFAULT 'moderate';
ALTER TABLE portfolios ADD COLUMN IF NOT EXISTS total_pnl NUMERIC(18,8) DEFAULT 0.0;
```

```bash
PGPASSWORD=PASSWORD psql -h 127.0.0.1 -p 15432 -U elson_user -d elson_db \
  -c "ALTER TABLE {table} ADD COLUMN IF NOT EXISTS {col} {type} DEFAULT {default};"
```

---

## Rollback Migration

```bash
cd backend && alembic downgrade -1
```
