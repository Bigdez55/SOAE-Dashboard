# AI Model Health Check — elson-finance-14b

Check the complete AI stack: VM, vLLM server, model, and EFT integration.

---

## Step 1 — vLLM VM Status

```bash
gcloud compute instances describe elson-dvora-training-l4-2 \
  --zone=us-west1-a \
  --project=elson-33a95 \
  --format="value(status,networkInterfaces[0].networkIP)"
```

Expected: `RUNNING` | `10.138.0.4`

## Step 2 — vLLM Server Ping (from local if VPN/SSH connected)

```bash
# Only works if on same VPC or with SSH tunnel
curl -s http://10.138.0.4:8000/health 2>/dev/null || echo "Not reachable from local (expected — VPC only)"
```

## Step 3 — API Health (includes vLLM reachability)

```bash
curl -s https://api.elsontrade.com/health | python3 -m json.tool
```

Check for `"vllm_status"` field in response.

## Step 4 — EFT Agent Config Inventory

```bash
# Count and list active EFT agent configs
grep -n "AgentConfig\|agent_id" backend/app/services/eft_agent_config.py | head -30
```

Expected: 14 agent configs.

## Step 5 — Fine-Tuning Data Inventory

```bash
# Count training examples and outcome coverage
python3 -c "
import sys
sys.path.insert(0, 'backend')
from app.services.model_evaluation_service import ModelEvaluationService
print('Checking training data...')
" 2>/dev/null || echo "Run from backend/ directory with venv activated"
```

Manual check via API:
```bash
TOKEN=$(curl -s -X POST https://api.elsontrade.com/api/v1/auth/login \
  -H "Content-Type: application/json" \
  -d '{"email":"claude.test@example.com","password":"ElsonTest2026!"}' \
  | python3 -c "import sys,json; print(json.load(sys.stdin).get('access_token',''))")

curl -s "https://api.elsontrade.com/api/v1/analytics/model-performance" \
  -H "Authorization: Bearer $TOKEN" | python3 -m json.tool 2>/dev/null | head -30
```

## Step 6 — Model Drift Indicators

```bash
# Recent AI vs rule-based signal ratio
curl -s "https://api.elsontrade.com/api/v1/analytics/signal-sources?days=7" \
  -H "Authorization: Bearer $TOKEN" | python3 -m json.tool 2>/dev/null
```
