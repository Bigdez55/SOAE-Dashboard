# APEX Dashboard Engineering System

## Overview

A system of **15 APEX agents** and **14 cross-cutting reusable skills** that together form a complete dashboard engineering platform. Each agent embodies domain expertise in their area; each skill provides framework-agnostic patterns reusable across all projects.

**Agents** live in `.claude/agents/` and are invoked explicitly. **Skills** live in `.claude/skills/` and are auto-discovered by Claude Code based on trigger keywords in conversation.

## Agent Collaboration Patterns

### Building a New Dashboard

```
dashboard-scaffold → [framework agent] → theme-engine → kpi-card-factory
                                        → chart-builder → table-master
                                        → data-pipeline → ai-insights
```

### Adding Export + Alerts

```
export-suite → chart-builder (PNG snapshots)
alert-system → ai-insights (anomaly-fed alerts)
```

### Performance Optimization

```
perf-profiler → [framework agent] (code splitting, lazy loading)
             → TURBO agent (full architecture)
```

### Testing Pipeline

```
test-harness → SENTINEL agent (full CI gates)
             → deploy-pipeline (test gates before deploy)
```

### Enterprise Security

```
auth-guard → VAULT agent (full enterprise architecture)
           → deploy-pipeline (certificate-based deploy auth)
```

## Usage Examples

### Trigger a Framework Agent
> "Build me a React dashboard with KPI cards and trend charts"
→ Activates **PRISM** (apex-react) + **kpi-card-factory** + **chart-builder**

### Trigger a Reusable Skill
> "Export this dashboard to PDF"
→ Activates **export-suite** skill

### Trigger an AI Analysis
> "What's the health score for this month's KPIs?"
→ Activates **ai-insights** skill

### Trigger Performance Optimization
> "The dashboard feels slow when filtering"
→ Activates **perf-profiler** skill + **TURBO** agent for deep optimization

### Trigger Testing Setup
> "Set up Jest and Playwright for this project"
→ Activates **test-harness** skill

### Trigger Deployment
> "Set up GitHub Actions to deploy to GitHub Pages"
→ Activates **deploy-pipeline** skill

## Model Assignment Strategy

- **Opus** (deep reasoning): Architecture, complex logic, AI integration, design systems
- **Sonnet** (fast execution): Code generation, scaffolding, testing, deployment

Agents specify their model in the YAML frontmatter of their agent file. Skills do NOT specify a model — they inherit from the conversation context.
