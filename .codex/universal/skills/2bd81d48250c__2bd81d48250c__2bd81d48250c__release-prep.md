# GEN.OS Release Preparation Protocol

Run at sprint close after all gates pass.

## Step 1 — Final Gate Sweep

```bash
# All must exit 0
python platform/check_language_policy.py
ruff check .
make -f Makefile.rfc002 run-tests
pytest -q
bandit -r platform/ ai/ -ll
trivy fs . --severity CRITICAL,HIGH --exit-code 1 \
  --skip-dirs .claude,node_modules,build/rootfs
```

Run `/compile-check` — confirm sprint target count (N/N).

## Step 2 — Changelog Update

File: `docs/CHANGELOG.md`

```markdown
## Sprint N — [YYYY-MM-DD]

### Added
- [Module A]: [description] (N files, N lines)
- [Module B]: [description]

### Changed
- [Component]: [what changed and why]

### Fixed
- [Bug ID]: [description] (found by [agent])

### Security
- [Finding ID]: [fix description]

### Compile Check Count
- Sprint N: N/N (total cumulative: N checks pass)
```

## Step 3 — Git Tag

```bash
# Verify clean working tree
git status
git diff --stat HEAD

# Sprint tag format (not semver for sprint gates)
git tag -a sprint-[N] \
  -m "Sprint [N] — [goal] ([N/N compile checks pass])"

# Verify
git show sprint-[N] --stat | head -20
```

## Step 4 — Push (confirm first)

```bash
# Show what will be pushed
git log origin/main..HEAD --oneline
git tag --list 'sprint-*' | tail -5

# Push (requires explicit confirmation)
git push origin main
git push origin sprint-[N]
```

## Step 5 — Agent Memory Update

After push, update:
1. Global MEMORY.md: `Sprint N COMPLETE (commit HASH): [summary]`
2. `master-orchestrator` memory: sprint synthesis doc
3. All involved agent MEMORY.md: sprint N findings

## Commit Message Format

```
feat: Sprint N — [goal description] (N/N compile checks pass)

Sprint N deliverables:
- [Module]: [N files, N lines — description]

Test gates: RFC-002 80/80, pytest N/N, cppcheck 0 warnings
Security: bandit 0 HIGH/CRITICAL, trivy 0 CVEs
```
