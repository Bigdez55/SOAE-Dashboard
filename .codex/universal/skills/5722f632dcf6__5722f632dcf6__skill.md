# Full Test Suite — Elson TB2

Run all test categories and report results. Fix failures before flagging as done.

---

## 1. Backend — pytest

```bash
cd backend && python -m pytest \
  --tb=short \
  -q \
  --ignore=tests/integration \
  2>&1 | tail -30
```

**Baseline:** 1101 passed, 4 skipped, 0 failures
Flag any regressions vs. baseline.

## 2. Backend — Integration Tests

```bash
cd backend && python -m pytest tests/integration/ \
  --tb=short -q 2>&1 | tail -20
```

Note: Requires live DB connection via Cloud SQL proxy. Skip if proxy not running.

## 3. Frontend — TypeScript

```bash
cd frontend && npx tsc --noEmit 2>&1 | head -40
```

**Baseline:** 0 errors. Known pattern: Recharts v3 formatter callbacks require `as any` cast — this is expected.

## 4. Frontend — ESLint

```bash
cd frontend && npx eslint src/ --ext .ts,.tsx --max-warnings=10 2>&1 | tail -20
```

## 5. Go Microservices — Build Check

```bash
cd backend/go/market-gateway && go build ./... 2>&1
cd backend/go/order-router && go build ./... 2>&1
cd backend/go/risk-engine && go build ./... 2>&1
```

All three must complete without errors.

## 6. Rust/WASM — Build Check

```bash
cd frontend/src/wasm && cargo check 2>&1 | tail -10
```

---

## Pre-Test Checklist

Before running:
- [ ] Redis running locally: `brew services list | grep redis`
- [ ] Virtual environment activated: `source backend/venv/bin/activate`
- [ ] PYTHONPATH set: `export PYTHONPATH=$PWD/backend`

## Results Format

```
BACKEND TESTS:    [X passed / Y failed / Z skipped]
INTEGRATION:      [PASS / SKIP / FAIL]
TYPESCRIPT:       [0 errors / N errors]
ESLINT:           [0 warnings / N warnings]
GO BUILDS:        [✅ market-gateway / ✅ order-router / ✅ risk-engine]
RUST/WASM:        [✅ OK / ❌ ERROR]

OVERALL: PASS ✅ / FAIL ❌
```

For any failures, show the specific error and the file/line causing it.
