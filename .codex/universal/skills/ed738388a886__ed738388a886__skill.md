# Pre-Deployment Readiness Check

Run all checks and report a GO / NO-GO verdict. **Do not deploy — checks only.**

---

## 1. Schema Drift Check

```bash
git diff HEAD~5 -- backend/app/models/ | grep "Column("
```

- ✅ No new columns = PASS
- ❌ New columns without ALTER TABLE = **NO-GO** (would crash production)

## 2. Requirements Sync

```bash
ls backend/requirements*.txt
```

> This project uses a single `requirements.txt` — Dockerfile references it directly. Verify no stray `requirements-docker.txt` has diverged.

## 3. Secret Inventory

```bash
gcloud secrets list --project=elson-33a95 --filter="name:elson" --format="table(name)"
```

Cross-reference with every `os.getenv(...)` call in backend:
```bash
grep -rh "os.getenv\|SECRET_KEY\|os.environ.get" backend/app/ | grep -v "#" | sort -u
```

## 4. TypeScript Check

```bash
cd frontend && npx tsc --noEmit 2>&1 | tail -20
```

## 5. Backend Tests

```bash
cd backend && python -m pytest --tb=short -q 2>&1 | tail -30
```

## 6. Linting

```bash
cd backend && python -m pylint app/ --fail-under=7.0 --errors-only 2>&1 | tail -20
cd frontend && npx eslint src/ --max-warnings=10 2>&1 | tail -20
```

## 7. Docker Build Test

```bash
cd backend && docker build -t elson-pre-deploy-check . --quiet 2>&1 | tail -5
```

## 8. Model Column Diff (compare models to DB schema)

```bash
# List current model columns
grep -rn "Column(" backend/app/models/ | grep -v "#"
```

Compare output to known deployed schema. Flag any new fields.

---

## Report Format

Provide a summary table:

| Gate | Status | Notes |
|------|--------|-------|
| Schema Drift | ✅/❌ | |
| Requirements | ✅/❌ | |
| Secrets | ✅/❌ | |
| TypeScript | ✅/❌ | |
| Tests | ✅/❌ | X passed, Y failed |
| Lint | ✅/❌ | |
| Docker Build | ✅/❌ | |

**Verdict: GO ✅ / NO-GO ❌**

If NO-GO, list blocking issues with remediation steps.
