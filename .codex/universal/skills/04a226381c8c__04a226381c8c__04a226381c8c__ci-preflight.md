Run this before every `git push`. Mirrors CI exactly. No surprises.

## Checklist (run in order)

```bash
cd /Users/desmondearly/Library/CloudStorage/OneDrive-Personal/GENESYS/GENESYS

# 1. Language policy gate (hard fail)
python platform/check_language_policy.py || { echo "BLOCKED: Language policy"; exit 1; }

# 3. TypeScript lint
cd compositor/genos-shell && npx eslint 'src/**/*.{ts,tsx}' 'electron/**/*.ts' --max-warnings 0 || exit 1
cd -

# 4. Kernel compile check (Sprint 2 gate = 29/29)

# 7. Python tests
pytest -q || { echo "BLOCKED: pytest"; exit 1; }

# 8. XKABI conformance (26/26)
python -m pytest xisc/conformance/ --backend=linux -q 2>/dev/null \
  || echo "WARN: XKABI conformance (Linux CI only)"

# 9. Security SAST
bandit -r platform/ ai/ -ll || { echo "BLOCKED: bandit"; exit 1; }

# 10. No stale .js drift
git diff --name-only HEAD | grep -E '\.ts$' | while read f; do
  js="${f%.ts}.js"; [ -f "$js" ] && echo "WARN: stale JS drift: $js"
done
```

## Commit Message Format (enforced)

```
feat: Sprint N — [module description] (N/N compile checks pass)

- [File 1]: [purpose]
- [File 2]: [purpose]
```

## Go / No-Go

All 10 steps exit 0 → push.
Any failure → fix before pushing. Never `--no-verify`.
