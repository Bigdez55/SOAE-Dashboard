# GEN.OS Full Test Matrix — Mandatory E2E Protocol

All 10 test types x 3 methodologies x 3 static analysis tools.
Required by memory policy before every sprint close.

## Methodology Mapping

| Test Type | White Box | Black Box | Grey Box |
|-----------|-----------|-----------|----------|
| Smoke | Trace boot path in kmain() | Run binary; check serial output | Check boot flags + output |
| Functional | Trace each function path | Call API; verify return | API + internal state check |
| Integration | Trace cross-module calls | System-level scenario test | API contracts + timing |
| Regression | Diff against Sprint N-1 | Re-run all known scenarios | Golden output comparison |
| Load | Instrument spinlock contention | Sustained request flood | Load + CPU/memory metrics |
| Stress | Push pool to exhaustion | Max connections + payload | Pool + OS resource limits |
| Security | Audit XKABI rights in source | Fuzz inputs + invalid ops | Boundary inputs + rights |
| UI | Paint path trace in compositor | Visual regression screenshots | Input event to visual delta |
| Fuzz | Source-level invalid input paths | AFL/libfuzzer on API surface | Structured invalid inputs |
| Reliability | Fault injection in source | Crash/restart cycle test | MTTR measurement |

## Test Runner Commands

```bash
# Foundation (Sprint 0)
make -f Makefile.foundation test

# RFC-002 (80 tests)
make -f Makefile.rfc002 run-tests

# XKABI Conformance (26 tests)
python -m pytest xisc/conformance/ --backend=linux -q

# Security barriers

./build/rfc002/xkabi_conformance

# See /static-analysis skill
```
