# GEN.OS QEMU Boot Test — Boot Verification for GEN.OS

Boot and verify GEN.OS in QEMU before flashing to HP EliteBook x360 hardware.

## When to use

- After modifying AetherBoot (stage1.asm), XENOS (main.c, entry.asm), or GENSD (main.c)
- After changes to `tools/iso/gen_iso.c` or the ISO build pipeline
- Before any hardware deployment to the HP EliteBook x360
- To reproduce a boot failure without physical hardware access

## Prerequisites

```bash
# Install QEMU and OVMF (UEFI firmware) on Ubuntu
sudo apt-get install -y --no-install-recommends \
  qemu-system-x86 ovmf

# OVMF firmware paths
OVMF_CODE=/usr/share/OVMF/OVMF_CODE.fd
OVMF_VARS=/usr/share/OVMF/OVMF_VARS.fd
```

## GEN.OS Machine Profile (HP EliteBook x360 simulation)

```bash
QEMU_MACHINE="-machine q35,accel=kvm:tcg \
  -cpu Skylake-Client,+x2apic,+pdpe1gb \
  -smp cores=4,threads=1 \
  -m 8G"
```

- `q35`: PCIe chipset (matches HP EliteBook x360 real topology)
- `Skylake-Client`: matches Intel Core i5-8xxx/i7-8xxx (HP EliteBook x360 CPU)
- `kvm:tcg`: use KVM if available, fall back to TCG (software emulation)
- `8G`: matches minimum EliteBook x360 config

## UEFI Boot (AetherBoot via OVMF)

```bash
# Build the ISO first
make -C tools/iso clean && make -C tools/iso

# Boot with UEFI
qemu-system-x86_64 \
  ${QEMU_MACHINE} \
  -drive if=pflash,format=raw,readonly=on,file=${OVMF_CODE} \
  -drive if=pflash,format=raw,file=${OVMF_VARS} \
  -cdrom build/gen-os.iso \
  -serial mon:stdio \
  -display none \
  -no-reboot \
  -d cpu_reset,guest_errors \
  -D /tmp/qemu-gen-os.log
```

Reference: QEMU x86-64 system emulation docs — `qemu.org/docs/master/system/target-i386.html`
OVMF (Open Virtual Machine Firmware) — `github.com/tianocore/edk2`

## Legacy BIOS Boot (LegacyForge / stage1.asm)

## Expected Boot Sequence

The GEN.OS boot follows `init/gensd/boot_chain.c` (Sprint 8):

```
Stage 1: kernel_ready  → XENOS kernel initialized (PAL-Aether wired)
Stage 2: storage       → XSTORE mounted
Stage 3: security      → XSEC initialized; SHA-256 KAT passed
Stage 4: network       → XNET stack up (tap0 or virtio-net)
Stage 5: packages      → XPKG index loaded
Stage 6: display       → XDISP DRM initialized (virtio-gpu or VGA)
Stage 7: compositor    → XCOMP running
Stage 8: shell         → XSHELL launched
Stage 9: ai            → XMIND weights loaded (if GGUF present)
Stage 10: done         → Boot complete
```

Verify in serial output:
```
[GENSD] boot_chain: stage kernel_ready — OK
[GENSD] boot_chain: stage storage — OK
[GENSD] boot_chain: SHA-256 KAT — PASS
...
[GENSD] boot_chain: done — GEN.OS ready
```

## QEMU Networking (XNET verification)

```bash
# Add virtio-net for network stack testing
-netdev user,id=net0,hostfwd=tcp::2222-:22 \
-device virtio-net-pci,netdev=net0
# XNET should acquire 10.0.2.15/24 via DHCP (QEMU default)
```

## QEMU Storage (XSTORE + NVMe)

```bash
# Add NVMe device for XSTORE testing
-drive file=/tmp/gen-os-nvme.img,if=none,id=nvme0,format=raw \
-device nvme,serial=deadbeef,drive=nvme0
# Create 1GB backing image: dd if=/dev/zero of=/tmp/gen-os-nvme.img bs=1M count=1024
```

## QEMU Display (XDISP + XCOMP)

```bash
# virtio-gpu for accelerated display (preferred)
-device virtio-gpu-pci \
-display gtk,zoom-to-fit=on

# VGA fallback (if XDISP DRM init fails)
-vga std
```

## Automated Boot Test

```bash
#!/bin/bash
# boot-test.sh — automated GEN.OS boot verification
set -e

TIMEOUT=60  # seconds

qemu-system-x86_64 \
  ${QEMU_MACHINE} \
  -cdrom build/gen-os.iso \
  -serial file:/tmp/gen-os-serial.log \
  -display none -no-reboot \
  -D /tmp/qemu.log &
QEMU_PID=$!

# Wait for boot complete signal
for i in $(seq 1 ${TIMEOUT}); do
  if grep -q "boot_chain: done" /tmp/gen-os-serial.log 2>/dev/null; then
    echo "BOOT OK — GEN.OS ready in ${i}s"
    kill ${QEMU_PID}
    exit 0
  fi
  sleep 1
done

echo "BOOT TIMEOUT after ${TIMEOUT}s"
kill ${QEMU_PID}
grep "FAIL\|PANIC\|ERROR" /tmp/gen-os-serial.log | head -20
exit 1
```

## Common Boot Failures

| Symptom | Cause | Fix |
|---------|-------|-----|
| Triple fault at entry | `entry.asm` GDT not loaded | Check `lgdt` in AetherBoot handoff |
| Stuck after kernel_ready | XSTORE init fails | Run WAL debug skill: verify XSTORE_OK |
| SHA-256 KAT FAIL | AES-GCM/SHA256 code corrupted | Re-run sprint3-xsec CI gate |
| XSHELL not launched | boot_chain.c `xchain_display_ready` false | Verify XDISP init returns XDISP_OK |
| QEMU black screen | XCOMP not writing to framebuffer | Switch to `-vga std` to isolate XDISP |
| Immediate reboot | Stack overflow in kernel | Check freestanding stack size (default 16KB) |

## HP EliteBook x360 Hardware Differences vs QEMU

| Feature | QEMU | HP EliteBook x360 |
|---------|------|-------------------|
| CPU topology | q35 simulated | Intel Core i5/i7-8xxx real HW |
| HPET | Emulated | Real HPET (14.318 MHz base) |
| xAPIC | xAPIC emulated | x2APIC (use x2APIC driver) |
| Storage | virtio-NVMe | PCIe NVMe (0x8086:real) |
| Display | virtio-gpu | Intel UHD 620 (0x8086:0x3EA0) |
| TPM | `-tpmdev passthrough` | TPM 2.0 embedded |

Always verify on QEMU before flashing. Use `hw-driver-verify` skill for hardware-specific checks.
